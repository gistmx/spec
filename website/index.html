# Clawware Blueprint

Five Layers

One document, five levels of understanding

Every .mx file contains the original markdown plus five comprehension layers — each designed for a different way of consuming information.

Layer 1

One-Liner

A single sentence. The tweet-length version. Max 280 characters. What you'd skim in a list of 200 files.

Layer 2

Core Gist

Three to five sentences. The elevator pitch. What you'd say if someone asked "what's this about?"

Layer 3

2-Minute Voice Brief

A conversational script purpose-written for audio. Not a summary read aloud — written for the ear, not the eye.

Layer 4

ELI5

Explain Like I'm 5. Simple words, fun analogies, wonder-preserving. Makes complex ideas feel magical.

Layer 5

Full Document

The original markdown, beautifully rendered with navigable sidebar. Because sometimes you need the whole thing — but now you have context before you dive in.

How It Works

Markdown goes in. Understanding comes out.

.mx is a superset of markdown. It's backward compatible — open it in any editor and it's just markdown with YAML frontmatter. But an MX-aware viewer unlocks the full experience.

1

Feed it markdown

Upload a .md file, paste a URL, point at a GitHub repo, or pipe it through the CLI.

2

AI reads and understands

Open-source language models generate five comprehension layers — each tuned for a different consumption context.

3

Voice is synthesized

The voice brief script is converted to natural audio using open-source TTS. Listen on the web, your phone, or download the MP3.

4

Get your .mx file

Download the enriched file, view it on a hosted link, share it with your team, or commit it alongside your .md in version control.

Open Format

The .mx specification

The format is open, documented, and backward compatible. Every .mx file is valid markdown. The comprehension layers are purely additive metadata in YAML frontmatter. Read the full spec on GitHub.

Use Cases

Who is this for?

Skill Publishers

You publish SKILL.md files. Generate .mx companions so developers can evaluate your skill in seconds instead of reading the full doc.

Open Source Maintainers

Your README is 2,000 lines. Give contributors a voice brief they can listen to while setting up their dev environment.

Technical Writers

Produce comprehension layers for every doc in your knowledge base. Non-technical stakeholders get the ELI5. Engineers get the full spec.

Teams

Point gist.mx at your repo. Get a project-level summary that synthesizes all your markdown files into one briefing with cross-file intelligence.
